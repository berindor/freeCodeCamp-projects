Packages

You should use packages to get access to specific letters (e.g. in the Hungarian language) and mathematical symbols and formulas. They are are also needed for tabular, enumeration and formatting (as we will see in the Space optimalization section).

I use the following packages for my exercise lists and exams created for my classes in Hungary.

  • for language:
    \usepackage[utf8]{inputenc}
    \usepackage[magyar]{babel}
    \usepackage{amsfonts}
    \usepackage{t1enc}
  • for numbering: \usepackage{enumerate}
    \usepackage{enumitem}
    \usepackage{theorem}
    \usepackage{hyperref}
  • for mathematical formula: \usepackage{amsmath}
    \usepackage{amssymb}
  • for space optimalization: \usepackage[a4paper,left=2cm,right=2cm,top=2cm,bottom=2cm]{geometry}
    \usepackage{multicol}
  • for tabular: \usepackage{array}
    \usepackage{tabularx}
Numbering

Usually there is a numbering of the exercises with arabic numbers, wich also refers to the section. I use sub-numbering for the task of the same type denoted by the letters of the alphabet.

I use the following enviroment before the beginning of the document for the numbering of the exercises with bold arabic numbers with a point after (e.g. 1.2.):

\theoremstyle{change}\theorembodyfont{\rm}\newtheorem{feladat}[tetel]{.}

Then an exercise can be written such that the code \begin{feladat} stands before the text of the exercise and the code \end{feladat} after it.

For the subtasks I create the following environment before the beginning of the document:

\newcommand{\abcd}[1]{\begin{enumerate}[label=\textbf{\alph*)}, nosep]#1\end{enumerate}}

Then I only have to use the code \abcd{} and put the subtask into the brakets with a code \item before each subtask. This creates bold letters with a closing bracket, e.g. a).

Space optimalization

When I create a list of exercises, I prefer to use as little paper as possible. Maybe this does not look nice, but I like this style.

In order to save space, I create a multi columns layout when the sub-tasks consists of short mathematical expressions. For a 3 column layout the code \begin{multicols}{3} is needed before the subtask enumeration and a code \end{multicols} after the enumeration. The number of columns depends on the size of the subtasks, I use usually 2, 3 or 4 columns.

Solutions

After creating the list of exercises it is nice to create also the solutions. The students are happy about that.

I recommend to put the solutions into a separate section, so they cannot be immediately viewed when starting a task. The numbering of the solutions can be synchronized with the number of the exercises using the hyperref package.

In order to create the references, first a label must be given to the exercise. An example of the beginning of an exercise with a label is shown below.

\begin{feladat}\label{f.vlsz.2dvv.3szog}

The number of the labelled exercise in this example can be referred as \ref{f.vlsz.2dvv.3szog}.

Review

After finishing the exercise list and the solutions, it is highly recommended to review your work. Creating solutions also helps you review whether the questions of tasks make any sense. It also helps you put the tasks in increasing order by difficulity. Then check the visibility of the mathematical formulas, if no index or letter is missing and everything means what you wanted to.
